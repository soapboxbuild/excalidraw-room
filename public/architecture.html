Sheet A-004 — Key Architectural Decisions

01 Auth strategy

Supabase SSR — server-side session via cookies

Use @supabase/ssr with createServerClient() in every Server Component and route handler. Middleware reads the cookie on every request and refreshes the token transparently. No localStorage, no client-side auth state.

Fix required: Drop the recursive RLS policy on organization_members. Replace with a SECURITY DEFINER function to break the cycle.

02 Data fetching

Server Components + Server Actions — no client waterfalls

Pages fetch data at request time via createServerClient() directly in the RSC. Mutations use Server Actions ('use server') so there's no API route boilerplate. AI streaming is the only exception — useChat from AI SDK handles the EventSource on the client.

03 Component system

shadcn/ui on Radix — no third-party component lock-in

shadcn copies source into the repo rather than wrapping a library. Tailwind CSS for all utility styling. CSS variables for org brand colours injected by the (app)/layout.tsx on every render from org.branding. Storybook at components.soapbox.build for component isolation and design review.

04 AI streaming

Vercel streaming response → EventSource → useChat

The Next.js app/api/chat/route.ts proxies to the Soapbox API and returns a ReadableStream. Vercel streams this via chunked transfer. The client uses useChat from @ai-sdk/react. Artifacts are rendered in a sandboxed <iframe> to prevent XSS from AI-generated HTML.

05 File upload + RAG indexing

Direct multipart to Soapbox API → chunked + embedded → pgvector search

Files are uploaded via the Soapbox API (not directly to Supabase Storage) so the server can chunk, embed, and index them immediately. Indexing status is streamed back to the client via EventSource (/api/indexing-status/:fileId) — the UI shows a live progress indicator per file. Retrieval uses pgvector nearest-neighbour search at query time, injected into the agent's context window.

Portfolio policies: docs uploaded here are available to ALL asset agents in the org via the shared portfolio RAG index. Per-asset docs are scoped to that asset's conversations only.
